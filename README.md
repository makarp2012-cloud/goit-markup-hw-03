# Домашнє завдання №3 · Верстка макета WebStudio

Цей проєкт є виконанням третього домашнього завдання, в якому відпрацьовано навички роботи з геометрією елементів, побудови сітки за допомогою Flexbox, створення контейнерів та правильного використання відступів (`margin` та `padding`).

## 📁 Структура проєкту

* `/images` — папка з оптимізованими зображеннями, взятими з макета.
* `/css` — папка з файлами стилів:
    * `modern-normalize.min.css` — підключена мініфікована версія нормалізатора стилів.
    * `styles.css` (або `common.css`) — основні стилі проєкту.
* `index.html` — розметка сторінки.

---

## 🛠️ Технічні вимоги та критерії якості

### Організація коду
* **Чисті назви:** У назвах файлів та папок повністю відсутня кирилиця і пробіли. Використовуються лише літери англійської мови, числа та дефіси/нижні підкреслення.
* **Форматування:** Весь вихідний код (HTML та CSS) відформатований за допомогою розширення **Prettier** відповідно до конвенції кодування.
* **Контент:** Усі зображення та текстовий контент точно відповідають наданому макету.

### Геометрія та Оформлення (CSS)
* **Скидання стилів:** Допускається глобальне скидання дефолтних стилів за селектором тегу для елементів `h1...h6`, `p` та `ul`.
* **Ефекти:** Тіні та інші декоративні ефекти навмисно не використовуються (це предмет наступних модулів). Уся робота зосереджена на геометрії та позиціонуванні.
* **Правило відступів:** * У елементів відсутні «пробиваючі» батьківський елемент зовнішні відступи.
    * Для вертикальних відступів між двома сусідніми елементами використовується `margin`.
    * Для проміжку між рамкою батьківського елемента і його дитиною використовується `padding`.
    * Всі розміри `margin` та `padding` задані точно за макетом.
* **Обмеження висоти:** У блоків відсутня фіксована висота. Вона плавно визначається їхнім контентом або внутрішніми падінгами.

### Сітка та Контейнери
* **Центрування:** Створено загальний допоміжний клас `.container` для центрування і обмеження контенту по ширині. 
* **Ширина контейнера:** Фіксована ширина дорівнює **1158px**. Клас `.container` знаходиться всередині хедера, футера та секцій, обгортаючи їхній контент.
* **Flexbox:** Елементи розташовані за допомогою Flexbox у тих місцях, де це необхідно (шапка, навігація, списки в секціях тощо), для забезпечення горизонтального вирівнювання.

### Особливості елементів сторінки
* **Хедер:** Висота хедера та посилань у ньому визначена висотою контенту та вертикальними падінгами. Додано тонку нижню рамку (border) відповідно до макета.
* **Секції:** Розташовані одна під одною як стопка книг, без зовнішніх відступів. Для всіх секцій використовується спільний клас `.section` з верхніми та нижніми падінгами по **120px**.
* **Приховування заголовків:** Для правильного (інклюзивного та доступного) приховування заголовка другої секції в CSS використано патерн `.sr-only`:

```css
.sr-only {
  position: absolute;
  white-space: nowrap;
  width: 1px;
  height: 1px;
  overflow: hidden;
  border: 0;
  padding: 0;
  clip: rect(0 0 0 0);
  clip-path: inset(100%);
  margin: -1px;
}
